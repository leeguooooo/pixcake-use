pixcake-use MVP

Read-only probes for building a PixCake parameter-level CLI.

目标

第一阶段不修改 PixCake 数据，不逆私有网络接口。MVP 只负责定位本地数据、生成快照、比较手动操作前后的变化，从而找出预设、项目和参数保存位置。

命令

命令用途
doctor检查 App、支持目录、数据库目录、偏好文件和运行进程。
snapshot生成文件和 SQLite 摘要快照。
diff比较两个快照，列出新增、删除、内容变化和 SQLite 表行数变化。
watch在一个短窗口内抓手动 UI 操作前后的变化。
schema查看单个 SQLite 数据库的表结构摘要。

参数探测流程

python3 -m pixcake_use snapshot --name before
# 在 PixCake 中只改一个参数，例如曝光 +0.50，然后保存新预设
python3 -m pixcake_use snapshot --name after
python3 -m pixcake_use diff snapshots/before.json snapshots/after.json

每次只改一个参数，重复曝光、对比度、高光、阴影等字段，就能建立 UI 文案到内部字段的映射。

边界

这个项目只做自有账号、自有图片的本地自动化。它不绕过登录、付费功能、额度、签名或服务端校验。
